Pure Pipes

  • Any pipe that doesnt take any arguments as an input is a pure pipe
  • Pure pipes are the default type of pipes in Angular.
  • They are only recalculated when the input to the pipe changes by reference.
  • This means that pure pipes do not run their transformation logic unless the input object or array changes in memory.
  • Characteristics of Pure Pipes:
    1. They are stateless and rely solely on their input arguments.
    2. They are called only when Angular detects a change in the input object’s reference.
    3. They are efficient and perform well since they don’t run on every change detection cycle.

Impure Pipes

  • Any pipe that takes any arguments as an input is a impure pipe.
  • Impure pipes are recalculated on every change detection cycle, regardless of whether the input object has changed
  • They are used for cases where the input might change in ways that are not detectable by Angular’s default change detection mechanism.
  • Characteristics of Impure Pipes
    1. They can be stateful and depend on more than just their input arguments.
    2. They run their transformation logic on every change detection cycle.
    3. They can be less efficient because they are recalculated more frequently.

Summary

    Pure Pipe:
  • Default type of pipe.
  • Recalculated only when input changes by reference.
  • Efficient and performant.
    Impure Pipes:
  • Recalculated on every change detection cycle.
  • Used for cases where inputs might change without reference changes.
  • Less efficient due to frequent recalculations.
